Task: Implement 'Nebula Bloom' Semantic Backgrounds

Goal: Create an animated, semi-transparent background layer that colors the space based on the semantic meaning of nearby posts (using embeddings).

1. The 'Semantic Cloud' Component:

Create a NebulaCloud component that renders large, overlapping Konva.Circle shapes.

Gradient Styling: Each cloud should use a fillRadialGradient that fades from a core color (e.g., #0000ff for blue) to transparent at the edges.

The Mapping Logic: >   - Write a helper embeddingToColor(embedding) that compresses the vector into an HSL color value.

Tip: Use the first 3 dimensions of the embedding to control Hue, Saturation, and Lightness.

2. Dynamic Background Layer:

Add a new Konva.Layer at the very bottom (below the stars).

For every cluster of stars, place one large NebulaCloud behind it. The color of this cloud should be the average color of the stars' embeddings in that cluster.

3. The 'Breathing' Animation:

Use Konva.Animation to create a slow "Swirl and Pulse" effect:

Rotation: Rotate the entire Nebula layer at a speed of 0.01 degrees per frame.

Scale: Subtly pulse the scale of the clouds between 1.0 and 1.2 over a 10-second loop to make the universe feel like it's "breathing."

4. Performance Optimization:

Set listening: false on the Nebula layer so it doesn't block mouse events for the stars.

Set filters={[Konva.Filters.Blur]} with a blurRadius of 50+ to make the clouds look soft and gaseous.

Use perfectDrawEnabled: false to ensure smooth 60FPS panning.

Output: Provide the NebulaLayer.jsx component and the logic to integrate it into the existing Universe stage.
